组织范式第三次洗牌——比拼的不是模型，而是你怎么编排「人+Agent」。

框架拆解 Iratang 智见 2026-06-02

如果你只读过一篇麦肯锡，我建议是 2025 年 9 月这篇 The Agentic Organization（智能体组织）。

不是因为它是「最新的 buzzword」，而是因为它把很多企业含糊其辞的 AI 转型，翻译成了一句硬话：继工业时代、数字时代之后，组织范式正在发生第三次迁移——未来公司，是「人 + AI Agent + 机器」在接近零边际成本下并肩工作的网络。

报告里有一个数字，我建议你抄在会议室白板上：

2～5 人的跨职能小队，已经可以监督 50～100 个专用 Agent，端到端跑完客户入职、产品上线或财务关账。

这不是 RPA 多加几个脚本，也不是给全员发 Copilot。这是运营系统、治理系统、人才系统、技术系统的同步重写。 下文基于公开报告要点展开，并补上对中国管理者的「可执行读法」。

一、为什么现在必须谈「智能体组织」？

麦肯锡把 AI 带来的变化，定义为自工业、数字革命以来最大的一次组织范式迁移。支撑这一判断的，是三条彼此加强的趋势：

  1. 任务长度在指数拉长。 2019 年以来，AI 可可靠完成的任务时长约每 7 个月翻倍；2024 年后约每 4 个月翻倍。报告撰写时，已约 2 小时；推演至 2027 年，可能出现连续 4 天无人值守仍保持质量的工作块。这意味着：「AI 只能做小事」的时间窗口，关闭得比多数董事会预想的快。
  2. Agent 部署光谱快速右移。 从增强现有活动 → 端到端工作流自动化 → AI-first 智能体系统，企业不是在「试用功能」，而是在迁移生产系统。
  3. 物理 Agent 进场。 无人机、自动驾驶、智能终端、早期人形机器人，让 AI 能作用于物理世界——组织设计不能再只考虑知识工作者。

换句话说：领先者赢的不是「谁买了更多 Copilot 席位」，而是谁先长出能编排 Agent 网络的组织神经系统。

二、五支柱：未来公司的「新骨架」

报告用五根支柱描述智能体组织（Exhibit 1）。我按「董事会听得懂」的语言重述如下。

支柱 1 · 商业模式：离客户更近，边际成本趋近「算力价」

三个子命题：

  • AI-native 渠道： 消费者绕过 App 与搜索，走向 AI 助手；企业需要 7×24 的「个人礼宾」式 Agent。欧洲某公用事业已为数百万客户提供多模态助手，显著缩短处理时长、提升满意度——这是前台革命。
  • AI-first 流程： 不是旧流程上贴自动化，而是先按 Agent 设计流程，再选择性把人放回环内或环上（above the loop）。银行、保险、电信已有 Agent 小队跑 KYC、理赔、客服——报告强调：这不是 usual automation。
  • 专有数据护城河： 公开互联网之外的行为流、产品使用、传感器数据，成为差异化燃料。AI 时代真正的 edge，来自「围墙花园」里的持续精炼。

支柱 2 · 运营模型：从职能金字塔到「成果导向 Agent 小队」

这是全文最核心的组织设计结论。

传统组织按职能竖井切分；数字时代用跨职能产品团队，但仍受「两个披萨团队」、邓巴数限制。智能体时代的基本单元，是 Agentic Team： 一小群多学科人类，拥有并监督底层 AI 工作流，对端到端业务成果负责（营销—产品—技术—数据—运营全链）。

麦肯锡在早期 adopters 中的经验数字是：2～5 人 可监督 50～100 个专用 Agent，跑完客户入职、产品上线、关账等流程。组织图将从「汇报线」转向 work chart——按任务与成果交换，而非仅按职位晋升。

警告同样尖锐：Agent 滥增生而没有上下文、导向与协同，是混乱之源。 赢家会构建扁平、高上下文共享的 Agentic 网络，且该网络可跨企业边界（新的 B2B 机会）。

支柱 3 · 治理：实时、嵌入式，人仍负最终责任

  • 决策加速： 年度预算节奏太慢；早期实践是「Agent 提议预算—情景 Agent 跑预测—报告 Agent 实时洞察」，财务负责人从「收表」变为「解读信号、压测情景」。
  • Agent 控 Agent： 类似 DevSecOps，批评 Agent、护栏 Agent、合规 Agent 嵌入流水线；从隐私到品牌声线，动作可实时记录与解释。
  • 人的问责不变，但形态变了： 合规官不再逐行审，而是定义政策、监控异常、调节人类介入强度。治理本身可能成为规模化瓶颈——oversight 能力决定 Agent 能放多快。

支柱 4 · 人才与文化：三种新画像 + 一种新「胶水」

画像核心能力在组织中的位置
M 型主管广谱 + AI 流利跨域编排 Agent 与混合劳动力
T 型专家深领域 + 系统设计重设计工作流、处理 Agent 失败例外
AI 增强一线业务共情 + 工具熟练销售/服务/HR/运营，少碰系统、多对人

文化要做两件事：编排粘合（对齐上下文与成果，识别并非一切都需要 Agentic AI）与伦理罗盘（避免为短期效率牺牲信任）。报告直言：赢家的分化点，在能否在高速转型中保持凝聚力与身份认同。

新概念 AI Fluency（AI 流利度）值得单独记住：不是会写 Prompt，而是理解 Agent 逻辑、敢授权、能信任并纠偏。

支柱 5 · 技术与数据：民主化 + Agent-to-Agent 协议

业务侧员工将能更独立地创建软件资产（早期 adopters 生产力至少 2×；报告甚至提到文学背景员工与工程师同等构建 Agent 工作流）。为支撑规模化，需要 Agentic AI mesh：可复用、带安全护栏的原子 Agent 与数据产品。

Agent-to-Agent 协议 被寄予厚望：系统之间用 Agent 对话，而非传统重型 API 集成——更快试验、更低互联成本。与此同时，动态采购 变得关键：锁死单一 LLM 厂商，可能数周即过时；架构须把 Agent 逻辑与供应商层分离。

三、早期信号：不是 PPT，是 P&L 前的组织实验

报告引用的案例，有一个共同模式：人不再填每一个格子，而是拥有端到端成果，并管理一支 Agent 队伍。

  • 某全球银行「Agent 工厂」：10 个 Agent 小队跑 KYC，质量与一致性显著提升；
  • 另一银行：人监督 Agent 小队现代化遗留核心，时间与工作量最多减 50%；
  • 欧洲车企与公共部门：Agent 逆向工程/改造遗留系统，人负责转向与验收。

这些案例与 Meta 5 月重组、英伟达 H-1B 逆势加码放在一张图里，会看到行业领先者已在三条路径上狂奔——买密度、换结构、建网络。

四、 executives 最常问「怎么开始」？报告给了一句警句

麦肯锡在高管访谈里听到最多的问题是：「How do I start?」 伴随的担心包括：愿景不清、数据/治理地基薄弱、试点泛滥、AI 到处开花却进不了 P&L、公关灾难。

报告用了一句略带幽默的警告：ending up with「more pilots than Lufthansa」——试点比汉莎航班还多，但利润表不动。

对中国企业的翻译是：先选一条端到端、价值可衡量、可行性高的主战场，用 Agentic Team 穿透，而不是再开一轮「AI 创新周」。

五、几点思考：把咨询语言变成管理动作

第一， 把 HR 系统升级为「人类 + Agent + 工作流」的资产账。绩效从「完成任务」转向「编排 Agent、交付成果的质量」。

第二， 为中层设计「环上管理者」角色：定目标、做取舍、处理例外、承担伦理仲裁——而不是「会用 ChatGPT 的专员」。

第三， 治理要前置，但别把 Agent 拽回人的审批速度；找到 oversight 的「甜蜜点」，否则规模化上限由合规排队决定。

第四， 技术采购从「选一个大模型」改为「建可更换的 Agent 架构」；上下文与专有数据才是长期壁垒。

写在最后

智能体组织不是「未来学」。它是对正在发生的组织现实的命名。当 2～5 个人可以驱动 50～100 个 Agent 时，管理者的核心价值，从「分配任务」迁移到「设计系统、设定边界、承担判断」。
